# `yujing` 目录规范

## 原则

1. 一个目录只承担一种职责。
2. 源码、构建产物、部署脚本、说明文档分开。
3. 双前端可以并存，但必须明确默认主线和候选主线。
4. 任何给微信开发者工具看的目录，都只放可导入产物，不放源码说明以外的临时文件。

## 目录职责

### `server/`
后端服务目录，只放 API、数据库、鉴权、LLM、脚本和后端文档。

### `miniapp/`
当前默认主线前端，基于 Uni-app。

- `src/`：源码
- `dist/build/mp-weixin/`：构建输出
- `devtools-ready/`：微信开发者工具导入目录

### `miniAPP-V2/`
候选主线前端，独立小程序工程。

- 作为完整工程保留
- 不引用 `miniapp/` 的构建产物
- 后续切主线时，只做显式迁移，不做“悄悄替换”

### `deploy/`
部署脚本、Nginx 配置和 ECS 说明。

### `docs/`
项目级规范、验收、联调、手册。

## 文件放置规则

- 源码放对应工程自己的 `src/`、`miniprogram/`、`server/src/`
- 构建产物放各自工程的产物目录
- 运行日志不进仓库
- 本地 `.env` 不进仓库
- 生产配置和密钥不进仓库

## 默认工作流

1. 后端改 `server/`
2. 当前联调用 `miniapp/`
3. 候选主线在 `miniAPP-V2/` 单独迭代
4. 需要演示或联调时，优先读 `README.md` 和 `HANDOFF.md`
